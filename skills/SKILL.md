```

❌ 过滤：顶部 3 个、右侧、底部推荐

```

```
- 已知的广告农场
- 低质量内容站

- 政府网站 (.gov.cn)
- 教育机构 (.edu)
- 主流媒体
- 知名科技公司
```

## 🧪 内容去毒功能 🆕

### 毒素识别

**信息毒素类型**：

| 类型 | 特征 | 处理方式 |
|------|------|---------|
| 🗑️ 垃圾信息 | 重复内容、关键词堆砌 | 过滤 |
| 📢 夸大宣传 | "震惊"、"必看"、"第一" | 降权 |
| 📰 虚假新闻 | 无来源、无作者、无时间 | 过滤 |
| 💊 医疗谣言 | 偏方、神药、治愈率 100% | 过滤 + 警告 |
| 💰 金融诈骗 | 高回报、稳赚、内幕 | 过滤 + 警告 |
| 🔞 不良内容 | 色情、暴力、赌博 | 过滤 |
| 🎭 深度伪造 | AI 生成虚假内容 | 标记 + 警告 |

### 去毒算法

**1. 来源验证**：
```python
可信来源 = [
    "政府网站", "主流媒体", "学术期刊",
    "知名公司官网", "权威机构"
]

if 来源 not in 可信来源：
    可信度评分 -= 30%
```

**2. 内容质量评估**：
```python
质量指标 = [
    "有无作者", "有无时间", "有无来源引用",
    "逻辑是否清晰", "数据是否可验证"
]

if 质量指标得分 < 60：
    标记为"低质量"
```

**3. 事实核查**：
```python
# 交叉验证多个来源
if 仅单一来源报道：
    标记为"待核实"

if 多个权威来源确认：
    可信度 += 50%
```

**4. 情绪分析**：
```python
if 包含极端情绪词汇 > 10 个：
    标记为"煽动性内容"
    可信度 -= 40%
```

### 警告系统

**警告级别**：

| 级别 | 标识 | 触发条件 | 处理 |
|------|------|---------|------|
| 🟢 可信 | ✅ | 权威来源 + 多方验证 | 正常显示 |
| 🟡 存疑 | ⚠️ | 单一来源/质量一般 | 标记显示 |
| 🟠 可疑 | ❗ | 低质量/夸大宣传 | 降权 + 警告 |
| 🔴 有毒 | 🚫 | 虚假信息/诈骗 | 过滤 + 警告 |

### 使用示例

**开启去毒搜索**：
```
"搜索疫苗信息（去毒）"
"search vaccine info (verified)"
"XXX 安全搜索"
"XXX 过滤虚假信息"
```

**去毒效果**：
```
原始结果：20 条
有毒内容：5 条（25%）
  - 虚假信息：2 条
  - 夸大宣传：2 条
  - 低质内容：1 条
过滤后：15 条（100% 可信内容）
```

## 🎯 搜索策略 v3.0

### 第一步：判断语言
- 包含中文字符 → 国内引擎
- 纯英文/其他拉丁字母 → 国外引擎
- 混合语言 → 根据主题判断

### 第二步：判断主题
- 国内公司/产品/政策 → 国内引擎
- 国际公司/技术/学术 → 国外引擎
- 新闻/时事 → 双引擎并行

### 第三步：时间检测
**自动识别时间关键词**：
- "今天"、"今日"、"today" → 2026-03-17
- "昨天"、"yesterday" → 2026-03-16
- "最近 24 小时"、"last 24h" → 过去 24 小时
- "本周"、"this week" → 本周
- "最新"、"latest" → 7 天内
- "本月"、"this month" → 30 天内

**无时间关键词** → 默认 30 天内

### 第四步：安全检测 🆕
```
1. 检测查询是否涉及敏感话题
   - 医疗健康 → 开启严格去毒
   - 金融投资 → 开启诈骗过滤
   - 新闻资讯 → 开启虚假新闻过滤

2. 设置过滤级别
   - 普通搜索 → 标准过滤
   - "去毒"、"安全" → 严格过滤
   - "学术"、"研究" → 权威源优先
```

### 第五步：选择引擎
**国内优先顺序**：
1. **百度搜索**（最大中文引擎）🆕
2. **360 搜索**（通用）
3. **360 新闻**（实时新闻）
4. **搜狗微信**（公众号文章）
5. **必应中文**（备用）
6. **今日头条**（实时热点）

**国外优先顺序**：
1. **Startpage**（Google 结果）🆕
2. **DuckDuckGo**（通用）
3. **DuckDuckGo News**（新闻专用）
4. **Qwant**（欧洲引擎）
5. **必应英文**（备用）
6. **Brave Search**（隐私保护）

### 第六步：多引擎聚合
**同时搜索 3 个引擎**：
```
主引擎：百度/Startpage
备用 1:360/Qwant
备用 2：必应
```

**合并策略**：
- 收集前 10 条结果
- 去重（基于 URL 和标题相似度）
- 过滤广告和有毒内容
- 按时间倒序排序
- 保留 5-8 条最佳结果

### 第七步：搜索 + 抓取
1. 同时搜索 3 个引擎
2. 合并去重结果
3. 过滤广告（自动识别）
4. 去毒处理（验证来源）
5. 选择最相关的 3-5 个 URL
6. 用 web_fetch 抓取详细内容
7. AI 生成摘要

## ⚠️ 注意事项

### 基础注意事项
1. **URL 编码** - 查询参数需要 URL 编码（空格用 `+` 或 `%20`）
2. **结果过滤** - 跳过"Sponsored"广告结果
3. **重试机制** - 如果首选引擎失败，自动切换到备用引擎
4. **结果数量** - 默认返回 5-10 条结果，可调整

### v3.0 新增注意事项

5. **时间格式** - 使用标准日期格式（2026-03-17）
6. **时区处理** - 默认 Asia/Shanghai 时区（GMT+8）
7. **实时性保证** - 新闻类查询优先使用新闻源
8. **多引擎超时** - 单个引擎超时 15 秒自动跳过
9. **结果去重** - 相似度>80% 视为重复
10. **缓存策略** - 相同查询缓存 30 分钟（实时查询除外）
11. **广告过滤** - 自动识别并过滤，无法关闭
12. **去毒级别** - 医疗/金融话题自动开启严格模式
